kcsd2d Class Constructor¶

The kcsd2d class constructor is a function that returns an instance of the kcsd2d class. Through the constructors arguments list we can input the data and our prior knowledge about the experiment. The syntax of the constructor looks as follows:

>>k = kcsd2d(el_pos, pots, ArgName, ArgVal, ...);

Variables el_pos and pots containing the electrode positions and recorded potentials are mandatory arguments and have to be provided in the presented order for the constructor to run. All other arguments are optional and can be given in arbitrary order in ArgName, ArgVal pairs. Values of most of those arguments are later available as class Properties.

Providing experimental data¶

The first argument (called by us el_pos) must contain the electrode positions. It must be a n x 2 matrix, where n denotes the number of electrodes and whose rows denote the (y, x) coordinates (y-vertical, x-horizontal) of the electrode positions.

The pots argument has to be a n x m matrix containing signals recorded at the corresponding n electrode positions of el_pos. m denotes the number of time frames analyzed.

Providing the experimental data is enough for the constructor to run. However, keep in mind that you may provide extra information by calling the constructor with optional arguments described below.

Defining the estimation area¶

By the estimation area we understand the positions at which the CSD is reconstructed. This area is set by default as:

>>[X, Y] = meshgrid(X_min: gdX: X_max, Y_min: gdY: Y_max);

where X_min, Y_min, X_max, Y_max are the extreme values of the electrode positions. gdX and gdY have default values:

>>gdX = 0.001*(X_max-X_min); gdY = 0.001*(Y_max-Y_min);

You may resign from the default and provide your own values by running the constructor with optional arguments:

>>k = kcsd2d(el_pos, pots, 'X_min', my_X_min, 'X_max', my_X_max, ...
  'Y_min', my_Y_min, 'Y_max', my_Y_max, 'gdX', my_gdX, 'gdY', my_gdY);

You can also define the estimation area explicitly, e.g.:

>>[my_X, my_Y] = meshgrid(0:0.01:1.4);
>>k = kcsd2d(el_pos, pots, 'X', my_X, 'Y', my_Y);

Model parameter options¶

Below are listed the model parameters whose values may also be provided in the constructor argument list:

Parameter Description Default value
R Initial value for basis function radii. If not provided, the default value is set to the minimal inter-electrode distance. Can be chosen by cross-validation at a later stage. depending on el_pos
h h parameter of the kCSD2d method. 1
sigma Conductivity.of extracellular space 1
lambda Lambda parameter for ridge regression. cross-validation

cross-validation - means that there is no default value. It is chosen by cross validation as described in Section 4.2 in our paper.

Layout of the sources¶

By default the base sources are distributed in the estimation area. This can be changed by choosing the options ext_X and ext_Y while calling the constructor. The overall number of sources can also be chosen arbitrarily.

Parameter Description Default value
ext_X margin by which the sources extend beyond the estimation area in the X direction. 0
ext_Y margin by which the sources extend beyond the estimation area in the Y direction. 0
n_src initial number of basis elements used. 300

Data management¶

Carrying out the kCSD method involves a lot of calculations. In order to avoid repeating the same calculations, the kcsd2d class stores certain results in mat-files in the kcsd2d_class/data directory and uses them later on. However, some users may prefer not to use this feature (e.g. e.g. due to the storage constraints). The feature can be turned off while calling the constructor by setting the value of the manage_data option to 0:

>>k = kcsd2d(el_pos, pots, 'manage_data', 0);

If you decide to use the default caching option, keep in mind that after some time the kcsd2d_class/data directory might get quite large. You might then consider removing some files.
